# FastAPI Backend Boilerplate

Production-grade **FastAPI** boilerplate for the VisaConnect / `globlejump` backend.

## Stack

| Concern        | Choice                                                             |
| -------------- | ------------------------------------------------------------------ |
| Web framework  | FastAPI (modular `APIRouter`, `lifespan`)                          |
| ORM            | SQLAlchemy 2.0 **async** (`asyncpg`)                               |
| Database       | PostgreSQL                                                         |
| Migrations     | Alembic (async `env.py`)                                          |
| Config         | `pydantic-settings` + `.env` + cached `get_settings()`             |
| Auth           | Hybrid — local OAuth2 password + JWT, **and** external identity-service JWT validation |
| Passwords      | `pwdlib` (Argon2)                                                  |
| Logging        | `structlog` (console in dev, JSON in prod) with request-ID binding |
| Metrics        | Prometheus (`/metrics`)                                            |
| Tracing        | OpenTelemetry (OTLP, optional)                                     |
| Error tracking | Sentry (optional)                                                  |
| Packaging      | `uv`                                                               |

## Project layout

```
app/
  core/        config, logging, security, exceptions, observability
  db/          Base, BaseModel (shared abstract model), async session
  models/      ORM models (User)
  schemas/     Pydantic request/response models
  services/    business logic / data access
  api/         deps (hybrid auth) + v1 routers (auth, users, health)
  middleware/  request-ID correlation
  main.py      app factory + lifespan
migrations/    Alembic (async)
tests/         pytest (async)
```

## Quick start (local)

```bash
# 1. Install dependencies
uv sync

# 2. Configure environment
cp .env.example .env
# generate a real secret:  openssl rand -hex 32  -> JWT_SECRET

# 3. Start PostgreSQL
docker compose up -d db

# 4. Apply migrations
uv run alembic upgrade head

# 5. Run the API (hot reload)
uv run uvicorn app.main:app --reload
```

Open http://localhost:8000/docs for Swagger UI.

## Smoke test

```bash
curl localhost:8000/health
curl localhost:8000/readiness

# Register
curl -X POST localhost:8000/api/v1/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"a@b.com","password":"supersecret","full_name":"Ada"}'

# Login (form-encoded, OAuth2 password flow)
TOKEN=$(curl -s -X POST localhost:8000/api/v1/auth/login \
  -d 'username=a@b.com&password=supersecret' | python -c 'import sys,json;print(json.load(sys.stdin)["access_token"])')

# Authenticated request
curl localhost:8000/api/v1/users/me -H "Authorization: Bearer $TOKEN"

# Metrics
curl localhost:8000/metrics
```

## Authentication model

* **Local tokens** are issued by `POST /api/v1/auth/login` (HS256, `iss = JWT_ISSUER`).
  These resolve to a real row in the `users` table.
* **External tokens** issued by the identity-service are accepted when
  `IDENTITY_JWT_SECRET` (HS256) or `IDENTITY_JWKS_URL` (RS256) is configured. They resolve
  to a `Principal` carrying only the user UUID — matching `BaseModel.created_by` /
  `updated_by`, which store bare UUIDs with no FK to the identity-service DB.

Use the `CurrentUser` dependency for endpoints that need a local account, or
`CurrentPrincipal` / `require_role(...)` for endpoints that accept either issuer.

## API response structure

Every JSON response uses a consistent envelope (defined in `app/schemas/response.py`).

**Success:**

```json
{
  "data": { "id": "…", "email": "ada@example.com", "role": "customer" },
  "meta": { "request_id": "8f1c…", "pagination": null }
}
```

**Error** (from `app/core/exceptions.py`):

```json
{
  "error": { "code": "conflict", "message": "A user with this email already exists" },
  "meta": { "request_id": "8f1c…" }
}
```

Endpoints return `ResponseEnvelope[T]` (a PEP 695 generic), so the OpenAPI schema stays
fully typed. The one exception is `POST /api/v1/auth/login`, which returns the bare
`{access_token, token_type}` to comply with OAuth2 (RFC 6749).

## Pagination

List endpoints use offset/page-based pagination via the `PaginationDep` dependency
(`?page=1&page_size=20`; `page_size` capped at 100). The `paginate()` helper returns
`(items, total)` and `page_meta()` builds the metadata block:

```
GET /api/v1/users?page=1&page_size=2     # admin only
```
```json
{
  "data": [ { "id": "…", "email": "…" }, { "id": "…", "email": "…" } ],
  "meta": {
    "request_id": "8f1c…",
    "pagination": {
      "page": 1, "page_size": 2, "total_items": 5,
      "total_pages": 3, "has_next": true, "has_previous": false
    }
  }
}
```

To paginate a new endpoint:

```python
from app.api.pagination import PaginationDep, paginate, page_meta
from app.schemas.response import ResponseEnvelope

@router.get("", response_model=ResponseEnvelope[list[ThingRead]])
async def list_things(params: PaginationDep, session: SessionDep, request_id: RequestIdDep):
    stmt = select(Thing).where(Thing.is_archived.is_(False)).order_by(Thing.created_at.desc())
    items, total = await paginate(session, stmt, params)
    return ResponseEnvelope[list[ThingRead]](
        data=[ThingRead.model_validate(t) for t in items],
        meta=page_meta(params, total, request_id),
    )
```

## Full stack via Docker

```bash
docker compose up --build
```

This starts PostgreSQL, runs a one-shot **`migrate`** service (`alembic upgrade head`),
then serves the API on port 8000.

Standalone image runs (Kubernetes / `docker run`) also auto-migrate via the
image entrypoint (`scripts/docker-entrypoint.sh`) unless you set
`RUN_MIGRATIONS=false`.

> Migrations are **not** applied during `docker build` — there is no database
> at build time. They apply on container start / CI deploy, same as Rails/Django.
